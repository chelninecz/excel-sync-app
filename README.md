# Генератор Excel-форм PPA из PDF-чертежей (Haier Style)

Приложение для автоматического создания форм одобрения деталей/материалов (PPA — Detail/Material Approval) на основе шаблона Excel и входного PDF-файла с чертежом.  
Программа автоматически извлекает технические требования и материал из PDF-документа (поддерживаются русский, английский и китайский языки), позволяет ввести дополнительную информацию о поставщике и детали, и генерирует готовый отчет, сохраняя оригинальное форматирование шаблона.

## 🚀 Основные возможности

- **Автоматическое извлечение данных:**
  - Распознавание текста из PDF с помощью pdfplumber (для цифровых PDF).
  - **RapidOCR** на базе ONNX для сканированных документов и чертежей с текстом в кривых.
  - Поддержка трёх языков: Русский, Китайский, Английский.
  - Автоматический поиск блока «Технические требования» (`Technical Requirements` / `技术要求`).
  - Извлечение информации о материале из штампа чертежа.

- **Умная обработка требований:**
  - Парсинг нумерованного списка требований с поддержкой различных форматов нумерации.
  - Геометрическая реконструкция текста: сортировка блоков по строкам с учётом отступов.
  - Автоматическое распределение требований по отдельным ячейкам шаблона: `{TECH_REQ_1}`, `{TECH_REQ_2}`, ... (до 50 пунктов).
  - Адаптивность: заполняются только доступные ячейки в шаблоне.

- **Работа с шаблоном Excel:**
  - Замена ключевых слов (`{SUPPLIER}`, `{PART_NAME}`, `{MATERIAL}` и др.) на введенные данные.
  - Полное сохранение форматирования исходного шаблона (шрифты, границы, цвета, логотипы).
  - Автоматическое включение переноса текста для длинных значений.

- **Интерфейс:**
  - Простой графический интерфейс (GUI) на Python (PyQt6).
  - Поля для ручного ввода: Поставщик, Название детали, Номер детали.
  - Поля для ручной коррекции автоматически распознанных данных (Материал, Требования).
  - Логирование всех операций в файл `ppa_generator.log`.

## 📋 Требования к системе

- **ОС:** Windows 10/11
- **Python:** 3.8 или выше
- **Poppler:** необходим для работы pdf2image (встроен в дистрибутив или устанавливается отдельно)

> **Примечание:** Tesseract OCR больше не требуется! Вместо него используется RapidOCR — современная OCR-библиотека на базе нейросетей ONNX, которая работает быстрее и точнее.

## 🛠️ Установка

### Шаг 1: Установка зависимостей Python

Откройте терминал (Command Prompt или PowerShell), перейдите в папку с проектом и выполните:

```bash
pip install -r requirements.txt
```

Необходимые библиотеки:
- `PyQt6` — графический интерфейс
- `pdfplumber` — извлечение текста из PDF
- `pdf2image` — конвертация PDF в изображения для OCR
- `openpyxl` — работа с Excel-шаблонами
- `numpy` — обработка изображений
- `rapidocr-onnxruntime` — OCR-движок на базе нейросетей

### Шаг 2: Установка Poppler (для pdf2image)

Библиотека `pdf2image` требует Poppler для конвертации PDF страниц в изображения.

**Вариант A: Использование встроенного Poppler**
Если вы создаёте исполняемый файл (например, через PyInstaller), поместите папку Poppler в директорию проекта:
```
project/
├── poppler/
│   └── Library/
│       └── bin/
├── haier_ppa_generator.py
└── requirements.txt
```

**Вариант B: Системная установка Poppler**
1. Скачайте последнюю версию: [poppler-xx.x.x.zip](https://github.com/oschwartz10612/poppler-windows/releases)
2. Распакуйте архив в удобное место (например, `C:\Program Files\poppler`)
3. Добавьте путь к папке `bin` в переменную среды `PATH`

## ⚙️ Подготовка шаблона Excel

Для корректной работы программы ваш файл-шаблон (`.xlsx`) должен содержать специальные метки в ячейках, которые программа будет заменять данными.

Используйте следующие ключевые слова внутри ячеек:

| Метка в шаблоне | Описание | Куда подставляется |
|---|---|---|
| `{SUPPLIER}` | Поставщик | Ввод пользователя |
| `{PART_NAME}` | Название детали | Ввод пользователя |
| `{PART_NUMBER}` | Номер детали | Ввод пользователя |
| `{MATERIAL}` | Материал | Авто-распознавание (или ручной ввод) |
| `{TECH_REQ_1}` ... `{TECH_REQ_50}` | Требование №1-50 | Пункты из списка PDF |
| `{TECH_REQ}` | Все требования одним текстом | Полный текст требований |

**Важно:** программа ищет эти строки внутри текста ячейки. Вы можете писать, например:
*«Проверить соответствие: `{TECH_REQ_1}`»*. Программа заменит только метку, оставив остальной текст и форматирование без изменений.

## 🚀 Как использовать

1. **Запустите приложение:**

```bash
python haier_ppa_generator.py
```

2. **Выбор файлов:**
   - Нажмите кнопку **«Выбрать PDF чертеж»** и укажите файл чертежа.
   - Нажмите кнопку **«Выбрать Excel шаблон»** и укажите файл `.xlsx`.

3. **Анализ PDF:**
   - Нажмите кнопку **«Анализировать PDF»**.
   - Программа попытается извлечь текст напрямую (через pdfplumber).
   - Если текст не найден или его мало (<300 символов), автоматически запустится RapidOCR для распознавания страниц как изображений.
   - Найденный материал и технические требования появятся в соответствующих полях.

4. **Проверка данных:**
   - Проверьте распознанные данные. При необходимости отредактируйте их вручную.
   - Введите: Поставщика, Название детали и Номер детали.

5. **Генерация:**
   - Нажмите кнопку **«Сгенерировать Excel»**.
   - Выберите место для сохранения готового файла.
   - Дождитесь сообщения об успешном создании файла.

## ⚠️ Обработка ошибок и ограничения

- **Не распознан текст:** если PDF является изображением очень низкого качества, RapidOCR может не распознать текст. В этом случае заполните поля вручную.
- **Мало требований:** если в чертеже найдено менее 5 пунктов, ячейки `{TECH_REQ_X}` сверх количества пунктов останутся пустыми.
- **Много требований:** поддерживается до 50 пунктов требований (`{TECH_REQ_1}` ... `{TECH_REQ_50}`). Если требований больше, будут взяты первые 50.
- **Формат PDF:** лучше всего работает с цифровыми PDF. Для сканов используется RapidOCR с геометрической реконструкцией текста.
- **Логирование:** все операции записываются в файл `ppa_generator.log` для отладки.

1. Установлены все зависимости из `requirements.txt`.
2. Poppler доступен в системе или находится в папке `poppler/` проекта.
3. В шаблоне Excel корректно прописаны ключевые слова `{...}`.
