# Генератор Excel-форм PPA из PDF-чертежей (Haier Style)

Приложение для автоматического создания форм одобрения деталей/материалов (PPA — Detail/Material Approval) на основе шаблона Excel и входного PDF-файла с чертежом.  
Программа автоматически извлекает технические требования и материал из PDF-документа (поддерживаются русский, английский и китайский языки), позволяет ввести дополнительную информацию о поставщике и детали, и генерирует готовый отчет, сохраняя оригинальное форматирование шаблона.

## 🚀 Основные возможности

- **Автоматическое извлечение данных:**
  - Распознавание текста из PDF (включая сканированные изображения через OCR).
  - Поддержка трёх языков: Русский, Китайский, Английский.
  - Автоматический поиск блока «Технические требования» (`Technical Requirements` / `技术要求`).
  - Извлечение информации о материале.

- **Умная обработка требований:**
  - Парсинг нумерованного списка требований.
  - Автоматическое распределение первых 5 пунктов по отдельным ячейкам шаблона: `{TECH_REQ_1}` ... `{TECH_REQ_5}`.
  - Адаптивность: если в шаблоне меньше 5 ячеек, заполняются только доступные; если требований меньше 5, остальные остаются пустыми.

- **Работа с шаблоном Excel:**
  - Замена ключевых слов (`{SUPPLIER}`, `{PART_NAME}`, `{MATERIAL}` и др.) на введенные данные.
  - Полное сохранение форматирования исходного шаблона (шрифты, границы, цвета, логотипы).

- **Интерфейс:**
  - Простой графический интерфейс (GUI) на Python (PyQt6).
  - Поля для ручного ввода: Поставщик, Название детали, Номер детали.
  - Поля для ручной коррекции автоматически распознанных данных (Материал, Требования).
  - Выбор папки для сохранения результата.

## 📋 Требования к системе

- **ОС:** Windows 10/11
- **Python:** 3.8 или выше
- **Tesseract OCR:** обязательно установлен в системе (для работы со сканированными документами)

## 🛠️ Установка

### Шаг 1: Установка Tesseract OCR

Для распознавания текста на изображениях необходим движок Tesseract.

1. Скачайте установщик для Windows: [Tesseract Installer](https://github.com/tesseract-ocr/tesseract/wiki/Download)
2. Рекомендуется версия `tesseract-ocr-w64-setup-5.x.x.exe`.
3. Запустите установщик.
4. **Важно:** на этапе выбора языков (`Additional language data`) обязательно отметьте:
   - Russian (`rus`)
   - English (`eng`)
   - Chinese (Simplified) (`chi_sim`)
5. Запомните путь установки (по умолчанию: `C:\Program Files\Tesseract-OCR`).
6. Добавьте путь к Tesseract в переменную среды `PATH` (рекомендуется) или укажите его в коде приложения.

### Шаг 2: Установка зависимостей Python

Откройте терминал (Command Prompt или PowerShell), перейдите в папку с проектом и выполните:

```bash
pip install -r requirements.txt
```

> **Примечание:** Если вы используете виртуальное окружение, активируйте его перед установкой.
> Если у вас нет файла `requirements.txt`, установите необходимые библиотеки вручную (например: `pypdf2`, `python-docx` / `openpyxl`, `pytesseract`, `PyQt6` и др.).

## ⚙️ Подготовка шаблона Excel

Для корректной работы программы ваш файл-шаблон (`.xlsx`) должен содержать специальные метки в ячейках, которые программа будет заменять данными.

Используйте следующие ключевые слова внутри ячеек:

| Метка в шаблоне | Описание | Куда подставляется |
|---|---|---|
| `{SUPPLIER}` | Поставщик | Ввод пользователя |
| `{PART_NAME}` | Название детали | Ввод пользователя |
| `{PART_NUMBER}` | Номер детали | Ввод пользователя |
| `{MATERIAL}` | Материал | Авто-распознавание (или ручной ввод) |
| `{TECH_REQ_1}` | Требование №1 | 1-й пункт из списка PDF |
| `{TECH_REQ_2}` | Требование №2 | 2-й пункт из списка PDF |
| `{TECH_REQ_3}` | Требование №3 | 3-й пункт из списка PDF |
| `{TECH_REQ_4}` | Требование №4 | 4-й пункт из списка PDF |
| `{TECH_REQ_5}` | Требование №5 | 5-й пункт из списка PDF |

**Важно:** программа ищет эти строки внутри текста ячейки. Вы можете писать, например:
*«Проверить соответствие: `{TECH_REQ_1}`»*. Программа заменит только метку, оставив остальной текст и форматирование без изменений.

## 🚀 Как использовать

1. **Запустите приложение:**

```bash
python main.py
```
*(Замените `main.py` на имя вашего основного файла скрипта.)*

2. **Выбор файлов:**
   - Нажмите кнопку **«Выбрать PDF»** и укажите файл чертежа.
   - Нажмите кнопку **«Выбрать шаблон Excel»** и укажите файл `.xlsx`.

3. **Заполнение данных:**
   - После выбора PDF программа попытается автоматически найти материал и технические требования. Они появятся в соответствующих полях.
   - Проверьте распознанные данные. При необходимости отредактируйте их вручную.
   - Введите: Поставщика, Название детали и Номер детали.

4. **Генерация:**
   - Нажмите кнопку **«Сгенерировать Excel»**.
   - Выберите папку для сохранения готового файла.
   - Дождитесь сообщения об успешном создании файла.

## ⚠️ Обработка ошибок и ограничения

- **Не распознан текст:** если PDF является изображением низкого качества или текст не найден, поля останутся пустыми. Заполните их вручную перед генерацией.
- **Мало требований:** если в чертеже найдено менее 5 пунктов, ячейки `{TECH_REQ_X}` сверх количества пунктов останутся пустыми.
- **Лишние требований:** если в чертеже более 5 пунктов, будут взяты только первые пять.
- **Формат PDF:** лучше всего работает с цифровыми PDF. Для сканов критически важна правильная установка Tesseract и языковых пакетов.
